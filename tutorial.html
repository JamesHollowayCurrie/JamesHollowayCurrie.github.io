Welcome to the tutorial section for Vulcan and C++. Here I will be posting and updating tutorials on the use of the Vulcan Graphics API for Windows using C++. Though all tutorials will focus on development in the windows environment, they can be easily adapted for writing applications on any platform. They will focus on the development of modern real-time rendering engines, but will be interspersed with tutorials covering basic C++, physics, game logic and other bits as supplement.

Who are these tutorials for?

These tutorials will focus on the use of Vulcan with C++ for the creation of various CGI effects and 3D rendering in a real-time context. As such, they will be of greatest interest to game developers and other CG professionals/students who work with real-time rendering solutions. The tutorials are written for the beginner, someone with little to no C++ or graphics experience, but many of the later tutorials will be of interest to seasoned graphics veterans as well.

What will you learn?

The tutorials start with a very basic introduction to C++ and simple 2D rendering with Vulcan. This will move into 3D rendering, deferred shading, screen-space effects, physics and interaction logic. The first few tutorials are listed in the order most appropriate for learning from scratch. However, each section has a 'start point' for which you can download the .cpp files and use them within the project created in the 'Getting Started' section.

Why Vulcan (as opposed to OpenGL or DirectX)?

Released in 2015 by the Kronos Group, Vulcan is the spiritual successor to OpenGL. It is designed to operate simultaneously on multiple types of GPUs, distributing loads seamlessly, for increased speeds, where OpenGL was built for workstation running a single discrete GPU. There are also a number of architectural improvements over OpenGL: It is able to run without data verification (OpenGL always verifies data and generates extensive error feedback), executes draw calls nearly 9 times faster by reducing the number of subroutine calls, and allows you to precompile Shaders into byte code.

DirectX is by far the most popular graphics API for video games and many graphics applications, but it is a proprietary Microsoft product. As such, it is only available on Windows and Microsoft licenced devices. The goal of these tutorials is to teach an open-source initiative in graphics, so they won't be covering any DirectX specific material. That said, much of these concepts transfer directly to any grpahics API, both now and in the future.
